# 文档导航

这里放的是 `wd` 里按业务能力拆开的专题文档。适合你已经明确自己要接哪一类能力，不想在总 README 里来回跳转的时候使用。

## 文档列表

- `auth.md`：微信小程序登录、支付宝小程序登录、`auth.UserHandler` 的职责边界、推荐接入流程
- `payment.md`：微信支付、支付宝支付、下单/退款/回调接入、业务侧 `Handler` 设计建议
- `message.md`：公众号消息、小程序订阅消息、企业微信机器人、阿里云短信的接入方式

## 如何选择

- 你在做登录体系或会员体系：先看 `auth.md`
- 你在做商城、订单、支付闭环：先看 `payment.md`
- 你在做通知、订阅、告警、短信：先看 `message.md`

## 与 README 的关系

仓库根目录 `README.md` 负责回答：

- 这个库整体有什么工具
- 基础设施层怎么接
- 各能力的大致入口在哪里

`docs/` 负责回答：

- 某一类能力的完整接入链路怎么落地
- 业务方需要实现哪些接口
- 默认会暴露哪些路由
- 更适合什么项目结构与职责分层
